# Example 01 · Support-triage agent (TypeScript)

> **Synthetic / teaching example.** Runs offline against the deterministic mock provider — no API key,
> no network. The numbers below are generated by the run and checked against `receipt.json`.

The simplest viable agent: **classify → route → evaluate**. A ticket goes to one LLM call that returns
a structured `{intent, urgency}` (validated with Zod); the agent routes it to `billing`, `technical`,
`account`, or escalates to `human-review` (basic HITL). It owns the **chaining → routing → evaluator**
pattern from [the pattern catalog](../../../docs/03-pattern-catalog.md).

## Run it

```bash
npm install
npm run run     # writes trace.jsonl, eval-report.json, receipt.json
npm test        # Vitest
```

Re-running is a no-op diff: the run is deterministic (canned fixtures, a synthetic clock — no `now()`,
no RNG).

## The receipt

The classification eval pass-rate is **100%** (**6/6** golden cases pass the routing gate of 0.8), and
the run emits **12** trace events conforming to
[`observability-event-schema.json`](../../../templates/observability-event-schema.json).

| Artifact | What it holds |
|----------|---------------|
| [`trace.jsonl`](trace.jsonl) | every `llm_call` + routing `decision`, one event per line |
| [`eval-report.json`](eval-report.json) | per-case results + the aggregate pass-rate |
| [`receipt.json`](receipt.json) | the headline numbers (machine-checked against this README) |

## How it works

1. `eval/golden.json` is the labeled golden set (6 tickets → expected intent + route).
2. `record-fixtures.ts` (off in CI) records the canned classifications, so the run is offline and the
   eval tests the **routing logic**, not the model.
3. `run.ts` classifies + routes each ticket, scores against the golden labels, and writes the receipts.
4. An unrecorded ticket throws `MissingFixtureError` — a drifted prompt fails loudly rather than
   silently hitting a real API.

See [chapter 02 — decision framework](../../../docs/02-decision-framework.md): this is a **workflow**
(you wire classify→route), not an open-ended agent loop.
